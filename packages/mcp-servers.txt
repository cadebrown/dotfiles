# mcp-servers.txt - MCP servers for Claude Code and Codex CLI
# Usage: install/claude.sh + install/codex.sh both read this file
#
# HTTP format:   <name> http <url> [auth=<source>] [extra args...]
# Stdio format:  <name> stdio cmd: <command...>
#
# Extra args on HTTP entries pass through to `claude mcp add`
# (e.g. --client-id <id>, --header "K: V"). Codex does not support these
# extras — install/codex.sh ignores them with a warning (servers using
# --client-id fall back to interactive OAuth on first Codex use).
#
# auth=<source> is a managed credential. Both Claude and Codex reconcile it
# on every sync. Sources:
#   auth=gh        — GitHub token via `gh auth token` (run `gh auth login` once).
#                    Claude: connection-time headersHelper (~/.claude/gh-mcp-headers.sh)
#                    Codex:  bearer_token_env_var = "GH_TOKEN", resolved by the
#                            codex() shell wrapper at launch — no token at rest
#   auth=context7  — CONTEXT7_API_KEY header from ~/.context7.env (optional:
#                    registers unauthenticated at lower rate limits when the env
#                    file is missing; create it with `bash install/auth.sh context7`)
#   auth=tavily    — Authorization: Bearer from ~/.tavily.env (TAVILY_API_KEY).
#                    Claude: header; Codex: bearer_token_env_var. Create with
#                    `bash install/auth.sh tavily`.
#   auth=exa       — x-api-key header from ~/.exa.env (EXA_API_KEY).
#                    Claude: header; Codex: env_http_headers. Create with
#                    `bash install/auth.sh exa`.
#   auth=gcloud    — Google ADC bearer for the official Google/Google Cloud
#                    remote MCP servers. No token at rest: a short-lived access
#                    token is minted at connection time from Application Default
#                    Credentials. Claude: headersHelper (~/.claude/gcloud-mcp-
#                    headers.sh) emitting Authorization + x-goog-user-project;
#                    Codex: bearer_token_env_var GOOGLE_MCP_TOKEN + env_http_
#                    headers x-goog-user-project, both filled by the codex()
#                    shell wrapper at launch. Set up with `bash install/auth.sh
#                    google` (one `gcloud auth application-default login`).
# Auth-managed entries cannot also pass --header / --client-id extras.
#
# URL placeholders: a {VAR} token in an HTTP url is replaced with $VAR (from
# ~/.<service>.env, sourced at install time) — for servers that carry the key
# in the URL itself (e.g. Firecrawl). Missing var → skipped on Claude, emitted
# inert with a warning on Codex.
#
# Install manually:
#   claude mcp add --transport <transport> --scope user <name> <url> [extra args...]
#   claude mcp add --scope user <name> -- <command...>

# Cloudflare: full API access (Pages, DNS, Workers, R2, Zero Trust, etc.)
# Write tools are gated by permissions.ask rules in home/dot_claude/settings.json.
cloudflare http https://mcp.cloudflare.com/mcp

# GitHub: PRs, issues, repo search, code review.
# OAuth/DCR is unsupported by GitHub's IdP, so we authenticate via gh CLI.
github http https://api.githubcopilot.com/mcp auth=gh

# OpenAI developer docs: API reference and guides for the OpenAI platform.
openaiDeveloperDocs http https://developers.openai.com/mcp

# Context7: up-to-date library documentation for any dependency.
# Remote streamable-HTTP endpoint (upstream-recommended over local npx stdio).
context7 http https://mcp.context7.com/mcp auth=context7

# DeepWiki: Devin-generated architecture wikis + Q&A for any public GitHub repo (no auth)
deepwiki http https://mcp.deepwiki.com/mcp

# Chrome DevTools: drive/inspect a live Chrome — network, console, performance
# traces. Google's official server; gracefully useless on machines without Chrome.
chrome-devtools stdio cmd: npx -y chrome-devtools-mcp@latest

# qmd: semantic+lexical search over the markdown knowledge layers (~/kb,
# Claude auto-memory, dotfiles docs). Talks to the warm local daemon
# (LaunchAgent dev.cade.qmd on macOS, lazy-start on Linux) — one model load
# shared by every agent session.
qmd http http://localhost:8181/mcp

# Blender: control Blender via MCP (requires addon.py installed in Blender — see github.com/ahujasid/blender-mcp)
blender stdio cmd: uvx blender-mcp

# === Web search & data retrieval (always-on) ===
# Tavily: LLM-tuned web search + extract + crawl + map in one server.
# 1000 credits/mo free (no card). Bearer auth (TAVILY_API_KEY).
tavily http https://mcp.tavily.com/mcp/ auth=tavily

# Exa: neural/semantic web search + crawling — best recall for research queries.
# x-api-key header (EXA_API_KEY).
exa http https://mcp.exa.ai/mcp auth=exa

# Firecrawl: best-in-class deep crawl + structured extraction (scrape, crawl,
# map, extract). Key rides in the URL path (FIRECRAWL_API_KEY).
firecrawl http https://mcp.firecrawl.dev/{FIRECRAWL_API_KEY}/v2/mcp

# === Google: official remote MCP servers (auth=gcloud / ADC) ===
# Google's own managed MCP endpoints (announced Dec 2025, expanded at Next '26).
# One `bash install/auth.sh google` (gcloud ADC login) authenticates all of
# them — no OAuth client to create, no local process, nothing at rest.
#
# Workspace servers are Developer Preview and READ-MOSTLY today: Gmail =
# search/read + drafts (no send); Calendar = read-only; Drive = read + create
# (no edit/delete). Enroll in the Workspace Developer Preview Program and run
# `gcloud services enable gmailmcp.googleapis.com drivemcp.googleapis.com
# calendarmcp.googleapis.com` (auth.sh google offers to do this). When these GA
# with full write, no change here is needed — same endpoints.
gmail       http https://gmailmcp.googleapis.com/mcp/v1    auth=gcloud
calendar    http https://calendarmcp.googleapis.com/mcp/v1 auth=gcloud
drive       http https://drivemcp.googleapis.com/mcp/v1    auth=gcloud
# Cloud servers are full read+write via the cloud-platform scope (IAM-governed).
# Run + Resource Manager cover nollege deploys; Storage + BigQuery for data.
# Needs: gcloud services enable run.googleapis.com cloudresourcemanager.
# googleapis.com storage.googleapis.com bigquery.googleapis.com.
cloud-run        http https://run.googleapis.com/mcp                   auth=gcloud
cloud-resmgr     http https://cloudresourcemanager.googleapis.com/mcp  auth=gcloud
cloud-storage    http https://storage.googleapis.com/storage/mcp       auth=gcloud
bigquery         http https://bigquery.googleapis.com/mcp              auth=gcloud
