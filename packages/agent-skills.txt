# agent-skills.txt - official/community agent skills for installed CLIs
# Usage: install/skills-sync.sh reads this file (overlay-extensible)
#
# Rows install into the single shared skill tree (~/.claude/skills, exposed
# to Codex/opencode/pi via the ~/.agents/skills symlink). Two row types:
#
#   <skill-dir> npx <owner/repo[#ref]> <skill-name>
#       → npx -y skills add '<owner/repo[#ref]>' --skill <skill-name> -a claude-code -g -y
#         (vercel-labs/skills CLI; lockfile at ~/.agents/.skill-lock.json —
#          moves to $XDG_STATE_HOME/skills/ if that var is set (ours isn't);
#          #ref pins a tag/branch/SHA — @name is a skill filter, NOT a ref)
#
#   <skill-dir> self <command...>
#       → the tool installs its own skill (run only when the binary exists)
#
# Idempotency: a row is skipped when ~/.claude/skills/<skill-dir>/SKILL.md
# exists. These dirs are INSTALLER-managed — never add chezmoi sources for
# them (one writer per skill dir; vendored skills live in home/dot_claude/).
